# OctoAcme Project Management Docs

This README provides an overview of OctoAcme's project management processes and links to all process documentation.

## Summary of OctoAcme Project Management Processes

OctoAcme follows an iterative, customer-first approach to project management built around five key phases: **Initiation**, **Planning**, **Execution**, **Release**, and **Retrospective**. Each project has a named Project Manager (PM) and Product Manager (PdM) who coordinate delivery, define outcomes, and measure success.

Key principles include delivering small, testable increments, maintaining clear ownership, making data-informed decisions, and fostering a culture of psychological safety and continuous improvement. Teams operate with a regular communication cadence — standups, weekly syncs, and monthly stakeholder updates — to keep delivery on track.

Risk management is built into each phase via a Risk Register, and stakeholder communication follows standardized templates for consistency. Releases are standardized by type (patch, minor, major) with pre-defined checklists, rollback plans, and deployment verification steps. After each sprint or milestone, the team holds retrospectives to capture learnings and drive continuous improvement.
